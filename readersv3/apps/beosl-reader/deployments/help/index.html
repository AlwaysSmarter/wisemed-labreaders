WiseMED BEOSL Reader

Ghid complet pentru importul fișierelor CSV BEOSL, configurarea directoarelor locale, verificarea trimiterii către WiseMED și depanarea loturilor de citiri.

CSV import File transport WiseMED API Istoric local

1. Ce face aplicația

BEOSL Reader monitorizează un director local pentru fișiere CSV, extrage valorile de interes pentru dozimetrie și le salvează în baza locală. După salvare, încearcă separat sincronizarea către WiseMED API. O eroare de API nu pierde datele deja importate local.

Import automat

Fișierele noi din import_dir sunt procesate și mutate în processed_dir sau failed_dir.

Console locală

Din UI poți vedea status, loguri, cereri importate și eventuale erori de sincronizare.

Sincronizare

Readerul trimite valorile către WiseMED folosind setările configurate în modulul wisemed-api.

3. Cum se face importul

  1. Pune fișierul CSV în directorul configurat la import_dir.
  2. Readerul detectează fișierul, îl parsează și extrage datele relevante.
  3. Rezultatele sunt salvate local în SQLite.
  4. Fișierul este mutat în processed_dir dacă importul a reușit sau în failed_dir dacă a apărut o eroare tehnică de parsare.

Fișierul este considerat procesat doar după parsarea locală. Problemele de sincronizare către WiseMED apar separat și se văd în loguri sau în statusul importului.

4. Formatul fișierului

Fișierul trebuie să fie CSV și să conțină coloanele folosite de protocolul BEOSL. În practică, cel puțin aceste câmpuri trebuie să existe:

  • Dosimeter Id
  • Hp10 Calc. Dose
  • Hp007 Calc. Dose
Dosimeter Id,Hp007 Calc. Dose,Hp10 Calc. Dose
100075100,0.121,0.085

5. Ce verifici în Settings

  • HTTP local address: dacă schimbi IP/port, după salvare te reconectezi la noul URL.
  • Update server: pentru update automat al aplicației.
  • Repeat mode: modul în care sunt tratate corecțiile pe loturi de rezultate.
  • Analytes: mapările de analize trebuie să fie corecte pentru importul local.

6. Troubleshooting

Fișierul nu este importat

  • verifică dacă extensia respectă pattern
  • verifică permisiunile pe directoarele locale
  • verifică dacă fișierul ajunge în failed_dir

Importul local merge, dar sincronizarea WiseMED eșuează

  • verifică setările din wisemed-api
  • verifică cheia cfg_wisemed_key
  • verifică mesajul exact din loguri

UI-ul local nu mai răspunde

  • verifică local_http.address
  • conectează-te la noul URL dacă portul a fost schimbat
