1. Ce face aplicatia

Barcode Printer este utilitarul WiseMED pentru tiparirea de etichete pe imprimante compatibile ZPL. Aplicatia expune un API local simplu, un endpoint legacy compatibil cu integrari mai vechi, o interfata web de administrare si suport pentru comenzi venite prin WiseMED WebSocket.

  • tiparire etichete din request HTTP sau WS
  • lista imprimantelor disponibile
  • pagina de setari pentru layout si selectie imprimanta
  • test print
  • istoric si statistici de tiparire
  • log al payload-ului ZPL trimis catre imprimanta pentru diagnostic

2. Meniul aplicatiei

Dashboard

Afiseaza statistica zilnica de tipariri, numarul total de etichete si numarul de joburi reusite sau esuate.

Setari

Configureaza imprimanta selectata, rezolutia, coordonatele elementelor, autentificarea locala si adresa HTTP pe care ruleaza aplicatia.

Istoric

Afiseaza joburile de tiparire salvate in baza locala, inclusiv IP client, codul tiparit, statusul si eventualele erori.

4. Exemple de tiparire

Request JSON

POST /api/barcode/print
Content-Type: application/json

{
  "bc": "TEST-93389",
  "pn": "TEST PRINT",
  "tc": "T",
  "no": "1"
}

Request legacy

GET /barcode/print?bc=TEST-93389&pn=TEST%20PRINT&tc=T&no=1

Campul principal de cod se poate transmite si ca bc, fileid sau code, in functie de integrarea existenta.

Cum configurezi imprimanta din Settings

  1. Deschide pagina Setari si identifica campul Selected printer.
  2. Daca imprimanta apare in sugestii, selecteaz-o direct.
  3. Daca imprimanta shared nu apare, introdu manual numele exact al printer-ului sau calea UNC: 
    \\server\printer
  4. Salveaza setarile din butonul Salveaza.
  5. Ruleaza imediat Test print pentru confirmare.

Campul de printer poate fi folosit acum atat ca lista de sugestii, cat si ca input manual pentru imprimante shared care nu sunt enumerate automat.

Cum interpretezi rezultatul

  • daca Test print merge, inseamna ca numele introdus este corect si contul curent are acces la imprimanta
  • daca numele se salveaza dar testul esueaza, problema nu este in UI, ci in accesul sistemului sau al contului de serviciu la acel printer
  • daca aplicatia ruleaza ca Windows Service, verifica imprimantele disponibile pentru contul serviciului, nu doar pentru utilizatorul conectat interactiv

6. Adresa HTTP locala

Aplicatia foloseste modulul comun local-http. Daca modifici HTTP local address din pagina de setari, listener-ul local este rebind-uit automat pe noua adresa dupa salvare.

  • nu este necesar restart manual al aplicatiei
  • dupa salvare trebuie doar sa te reconectezi la noul URL
  • setarea este persistata atat in local_http, cat si in modules.local-http
Exemplu:
127.0.0.1:8010
0.0.0.0:8010
192.168.1.50:8010

7. Cum este generat ZPL-ul

Payload-ul ZPL este compus din blocuri standard:

  • ^XA inceput eticheta
  • ^PW... latime tiparire
  • ^FO... pozitionare campuri
  • ^BY... parametri cod de bare
  • ^FD... date efective
  • ^FS sfarsit camp
  • ^XZ sfarsit eticheta

Pentru ZPL valid, comenzile de pozitionare trebuie sa fie ^FO, nu ^F0 sau alte variante corupte. Daca payload-ul este gresit, imprimanta poate ignora jobul sau poate afisa simptome neasteptate in spooler.

8. Debug si loguri

Pentru fiecare job, aplicatia poate salva:

  • IP-ul clientului care a cerut tiparirea
  • codul tiparit si numele pacientului
  • tipul de cod de bare
  • imprimanta folosita
  • status ok sau fail

In plus, logul runtime include payload-ul ZPL trimis catre imprimanta, inclusiv varianta Base64 pentru comparatie byte-cu-byte.

Acest lucru este util cand vrei sa compari exact payload-ul generat de aplicatie cu un payload trimis manual care functioneaza corect.

9. Comportament pe platforme

PlatformaComportament
WindowsPayload-ul este trimis raw catre spoolerul de imprimare prin API-ul WinSpool WritePrinter.
LinuxSe incearca lp -o raw, apoi fallback pe lpr -l.
macOSSe foloseste acelasi model raw prin lp sau lpr.

Scopul este ca imprimanta Zebra sa primeasca bytes ZPL direct, nu document text formatat de sistemul de printare.

10. Imprimante shared pe Windows

Aplicatia poate folosi si imprimante shared din sistem, dar trebuie facuta diferenta intre imprimantele vizibile pentru utilizatorul curent si imprimantele vizibile pentru contul sub care ruleaza serviciul.

  • daca aplicatia ruleaza interactiv, vede imprimantele instalate pentru acel utilizator Windows
  • daca aplicatia ruleaza ca Windows Service, vede imprimantele disponibile pentru contul serviciului, nu neaparat pe cele instalate doar in profilul utilizatorului tau
  • daca imprimanta shared nu apare in lista, poti introduce manual numele exact al queue-ului, de exemplu \\printserver\zebra_lab
  • daca numele este acceptat dar printul tot esueaza, de regula contul serviciului nu are acces la acea conexiune de printer

Faptul ca vezi imprimanta in Windows pe sesiunea ta nu garanteaza ca o vede si serviciul. Pentru servicii, imprimanta trebuie sa fie accesibila si pentru contul efectiv de rulare.

11. Troubleshooting

Spoolerul creeaza multe etichete, dar imprimanta nu incepe efectiv printul

  • verifica daca payload-ul ZPL este valid
  • verifica daca jobul ajunge raw, nu ca document text
  • compara payload-ul logat cu cel trimis manual care functioneaza

Nu apare imprimanta in lista

  • verifica daca imprimanta este instalata in sistemul de operare
  • pe Windows verifica daca serviciul de printare este activ
  • daca aplicatia ruleaza ca serviciu, verifica daca imprimanta este disponibila pentru contul serviciului
  • introdu manual numele queue-ului shared daca nu apare in sugestii
  • pe Linux/macOS verifica daca lpstat vede imprimanta

Tiparirea esueaza cu eroare

  • verifica numele exact al imprimantei selectate
  • verifica permisiunile utilizatorului sub care ruleaza serviciul
  • verifica jurnalul aplicatiei si istoricul de joburi

Nu mai poti accesa UI-ul dupa schimbarea portului

  • conecteaza-te la noua adresa configurata
  • daca ai ales un IP invalid, corecteaza deployments/config.yaml

12. Bune practici

  • foloseste valori in mm coerente cu dimensiunea reala a etichetei
  • testeaza intotdeauna cu Test print dupa schimbarea layout-ului
  • pastreaza o imprimanta default corect configurata daca nu vrei selectie explicita
  • nu salva valori contradictorii intre DPI si coordonate
  • foloseste istoricul si logurile ZPL pentru orice problema de randare
